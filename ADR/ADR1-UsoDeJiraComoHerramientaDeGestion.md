# Uso de jira como herramienta de gestión.

* Estado: aceptada.<!-- opcional -->
* Responsables: Alberto Martínez, Daniela Carrasco, Iván Navascues
* Fecha: 04/02/2026
## Contexto y Planteamiento del Problema

Necesitamos una herramienta para la gestión del proyecto. Debe ser capáz de crear tareas y asignarlas a los varios miembros del grupo.

## Factores en la Decisión <!-- opcional -->

* Familiaridad con la herramienta.
* Facilidad de uso.

## Opciones Consideradas

* Github
* Jira
* Notion

## Decisión

Hemos elegido Jira ya que la mayoría del grupo está familiarizado con la herramienta. Esta cumple con todos los requisitos que buscabamos y su versión gratuita es más que suficiente para llevar a cabo este proyecto.


## Ventajas y Desventajas de las opciones

### Github

* Positivo. La herramienta más usada con una gran cantidad de documentación.
* Positivo. Una única herramienta para la gestión del proyecto y la gestión del versionado.
* Negativo. Falta de experiencia en el equipo con las capacidades de gestión de github.
* Negativo. Interfaz poco intuitiva.

### Jira

* Positivo. Mucha experiencia por parte del equipo.
* Positivo. Interfaz entendible para un principiante.
* Negativo. Versión gratuíta algo limitante.

### Notion

* Positivo. Interfaz amigable.
* Negativo. Poca experiencia por parte del grupo.
* Negativo. Falta de flexibilidad. Integración complicada con github.
