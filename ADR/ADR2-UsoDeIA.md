# Uso de IA en el proyecto.

* Estado: aceptada.
* Responsables: Alberto Martínez, Daniela Carrasco, Iván Navascues
* Fecha: 04/02/2026
## Contexto y Planteamiento del Problema

Se plantean las herramientas IA disponbles para el desarrollo de este proyecto y cual sería su alcance.

## Factores en la Decisión 

* Confiabilidad.
* Facilidad de uso.
* Licenciamiento y compatibilidad con VS Code (editor a usarse por el equipo)

## Opciones Consideradas

* ChatGPT
* Github Copilot 
* Claude Code 

## Decisión

Se ha decido utilizar Github Copilot para sugerencia de código, revisión de errores y ejecutar pruebas tempranas dentro del desarrollo.


## Ventajas y Desventajas de las opciones

### ChatGPT

* Positivo. La herramienta es popular y cuenta con un número de consultas gratuitas
* Positivo. Puede ser útil para redactar texto como tutoriales o resúmenes.
* Negativo. No es netamente una herramienta para programación, por lo que puede generar mucho ruido al pedir segurencias de código o corrección de errores.
* Negativo. Si se requiere un modelo IA más robusto requiere suscripción.

### Github Copilot 

* Positivo. Reduce el tiempo en tareas repetitivas.
* Positivo. Es una herramienta netamente para programación y tiene compatiblidad con VS Code.
* Positivo. Se cuenta con licenciamiento para usuarios UPNA.
* Negativo. Riesgo de seguridad al cargarse los archivos como contexto.

### Claude Code 

* Positivo. Ofrece respuestas más extensas y mejor estructuradas en tareas de diseño y documentación.
* Negativo. Tiene ciertas limitaciones en VS code al no tener integración nativa como Copilot.
* Negativo. Modelo de suscripción
